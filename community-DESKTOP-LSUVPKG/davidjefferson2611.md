## Introdução

Git é um sistema de controle de versão distribuído, enquanto GitHub é uma plataforma de hospedagem de código-fonte que utiliza Git. Com Git e GitHub, você pode versionar seu código, colaborar com outros desenvolvedores e gerenciar projetos de maneira eficiente.

## Instalação

### Git

Para instalar o Git, siga as instruções abaixo de acordo com seu sistema operacional:

#### Windows

1. Baixe o instalador do Git no [site oficial](https://git-scm.com/download/win).
2. Execute o instalador e siga as instruções na tela.

#### macOS

1. Abra o Terminal.
2. Instale o Git usando Homebrew: `brew install git`.

#### Linux

1. Abra o Terminal.
2. Instale o Git usando o gerenciador de pacotes da sua distribuição. Por exemplo, no Ubuntu: `sudo apt-get install git`.

### GitHub

1. Acesse o [site do GitHub](https://github.com/).
2. Crie uma conta ou faça login se já tiver uma.

## Configuração Inicial

Depois de instalar o Git, configure seu nome de usuário e e-mail:

```bash
git config --global user.name "David Jefferson"
git config --global user.email "seuemail@exemplo.com"

Comandos Básicos do Git
Inicializar um Repositório
Para criar um novo repositório Git:
git init

Clonar um Repositório
Para clonar um repositório existente:
git clone <url-do-repositório>

Status
Para verificar o status do seu repositório:
git status

Adicionar Arquivos
Para adicionar arquivos ao staging area:
git add <arquivo>

Para adicionar todos os arquivos modificados:
git add .

Commit
Para criar um commit com suas alterações:
git commit -m "mensagem do commit"

Histórico de Commits
Para visualizar o histórico de commits:
git log

Trabalhando com Repositórios Remotos
Adicionar um Repositório Remoto
Para adicionar um repositório remoto:
git remote add origin <url-do-repositório>

Push
Para enviar suas alterações para o repositório remoto:
git push origin main

Pull
Para obter as alterações do repositório remoto:
git pull origin main
