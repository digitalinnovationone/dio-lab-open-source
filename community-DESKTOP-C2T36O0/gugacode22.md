## Minhas Contribuições
Durantes o curso de versionamento de código (GIT) da dio, eu descobri um comando através da documentação do git https://git-scm.com/book/pt-br/v2/Branches-no-Git-Rebase que não é comentado durante as aulas do curso. O git pull --rebase origin main corrige um erro  do repositório remoto com relação a commits que não estão no repositório local da sua máquina.

Exemplo:

git remote add origin + URL

! [rejected]        main -> main (non-fast-forward)
error: failed to push some refs to 

git pull --rebase origin main

Successfully rebased and updated refs/heads/main.

* (git pull --rebase origin main) esse comando busca mudanças mais recentes do repositório remoto e replica seus commits locais em cima da versões mais recentes do repositório remoto.
