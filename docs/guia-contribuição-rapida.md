# ⚡ Guia de Contribuição Rápida (Quick Start)

Este documento resolve o problema de complexidade para novos contribuidores que desejam realizar seu primeiro Pull Request neste repositório de forma ágil e padronizada.

## 🛠️ Problema Resolvido
Identifiquei que o processo de contribuição pode ser confuso para iniciantes devido à dispersão das informações. Este guia unifica os passos essenciais para garantir entregas de qualidade técnica.

## 🚀 Fluxo de Trabalho Otimizado

1. *Fork & Sync:* Certifique-se de que seu fork está atualizado com a main original antes de iniciar.
2. *Branch Semântica:* Crie uma branch com nomes claros:
   - feat/nome-da-melhoria para novas funções.
   - docs/ajuste-documentacao para melhorias de texto.
3. *Padrão de Commit:* Utilize [Conventional Commits](https://www.conventionalcommits.org/en/v1.0.0/):
   - Exemplo: feat: adiciona checklist de validação de código.
4. *Verificação Pré-PR:* Antes de abrir o Pull Request, verifique se não há links quebrados no Markdown.

## 📑 Checklist de Qualidade
- [ ] O arquivo foi salvo na pasta correta?
- [ ] A mensagem de commit explica o "porquê" da mudança?
- [ ] O link para o LinkedIn foi testado?
