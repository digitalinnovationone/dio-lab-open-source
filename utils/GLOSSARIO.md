# 📖 GLOSSÁRIO GIT & GITHUB PARA INICIANTES

Este guia foi criado para ajudar quem está a dar os primeiros passos no mundo Open Source a entender os termos técnicos mais comuns através de analogias simples do dia a dia.

---

## 🏗️ CONCEITOS INICIAIS (PREPARANDO O TERRENO)

| TERMO | METÁFORA SIMPLES | O QUE SIGNIFICA NA PRÁTICA? |
| :--- | :--- | :--- |
| **Repositório (Repo)** | Uma **Pasta de Arquivos** digital. | É o local (nuvem ou computador) onde todos os arquivos, pastas e o histórico do seu projeto ficam guardados. |
| **Fork** | Tirar uma **Fotocópia** do projeto de outra pessoa. | Cria uma cópia idêntica do repositório original na sua conta do GitHub, permitindo que faça alterações sem estragar o projeto principal. |
| **Clone** | **Descarregar** o arquivo para o seu computador. | Transfere a cópia do projeto que está na internet (GitHub) diretamente para a sua máquina local para que possa trabalhar nele. |

---

## 🛠️ CONCEITOS DE TRABALHO (MÃO NA MASSA)

| TERMO | METÁFORA SIMPLES | O QUE SIGNIFICA NA PRÁTICA? |
| :--- | :--- | :--- |
| **Branch (Ramificação)** | Uma **Linha do Tempo Alternativa** ou rascunho. | Uma bifurcação no código para testar uma nova funcionalidade (como criar o seu perfil) sem mexer na versão principal que já está a funcionar (`main`). |
| **Staging Area (`git add`)** | A **Mesa de Embalagem** dos correios. | O local onde coloca e organiza os arquivos que acabou de modificar, preparando-os antes de os enviar definitivamente para o sistema. |
| **Commit (`git commit`)** | Tirar uma **Foto com legenda** do seu progresso. | Grava oficialmente as alterações que estavam na mesa de embalagem. Cada foto precisa de uma mensagem curta explicativa (usando a Convenção de Commits). |

---

## 🚀 CONCEITOS DE ENVIO (COMPARTILHANDO COM O MUNDO)

| TERMO | METÁFORA SIMPLES | O QUE SIGNIFICA NA PRÁTICA? |
| :--- | :--- | :--- |
| **Push (`git push`)** | **Despachar a encomenda** pelos correios. | Envia os commits (as fotos com as alterações gravadas) do seu computador local de volta para o seu repositório remoto no site do GitHub. |
| **Pull Request (PR)** | **Pedir permissão** para colar o seu trabalho no painel da escola. | Uma solicitação oficial que faz aos donos do projeto original, pedindo para eles analisarem as suas alterações e adicioná-las ao projeto principal. |
| **Merge** | **Aprovar e juntar** as folhas no caderno. | A ação realizada pelos administradores do projeto para aceitar o seu Pull Request, unindo o seu código modificado ao código principal de forma definitiva. |
